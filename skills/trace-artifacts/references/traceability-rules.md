# Traceability Rules

Severity:

- `blocker`: conflicting policy/business behavior, missing critical requirement, or unsupported money/security/compliance interaction
- `major`: missing acceptance criterion, failure path, dependency, or requirement coverage
- `minor`: metadata, naming, or non-behavioral wording mismatch

Pass only when there are no blockers, every requirement is covered by its authoritative artifact, every behavioral requirement has acceptance coverage, and every diagram interaction is sourced or explicitly assumed.
