# Ticket Quality Gate

A final ticket must use the installed Change Request template and contain artifact metadata, a clear outcome, source-backed behavior, and observable acceptance criteria. It must contain `Context`, `Requirement`, and `Acceptance Criteria` in order, with exactly one Requirement variant: `Product`, `Back-end & Front-end Technical`, or `Configuration`.

Block final status when a required template section is absent or when an unanswered question can change data, permissions, money movement, compliance, API compatibility, or user-visible behavior.

Skip irrelevant Long Version prompts rather than writing `N/A`. Include dependencies, risks, assumptions, and open questions only when relevant to the selected Requirement variant.
