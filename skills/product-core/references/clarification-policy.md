# Clarification Policy

Ask in Vietnamese when missing or ambiguous information can change scope, user behavior, business rules, data, permissions, money movement, compliance, API compatibility, acceptance criteria, or system interactions.

## Process

1. Recall existing project context first. Never ask for information already available in an active cited source.
2. Separate blocking questions from non-blocking details.
3. Ask only the smallest set needed to continue, normally no more than five questions per round.
4. Explain briefly why each answer matters when the impact is not obvious.
5. Offer concrete options when the valid choices are known and identify the recommended default.
6. Use Vietnamese for the question and explanation; preserve English technical terms, field names, API names, and identifiers.
7. Wait for answers before producing a final artifact. A partial analysis may be shown, but label it incomplete.
8. If the user explicitly asks to continue without answering, document each assumption and mark the result `DRAFT — ASSUMPTIONS NOT CONFIRMED`.

Do not ask preference questions that do not materially affect the result. Do not repeat answered questions. Never silently turn missing information into a business rule.
